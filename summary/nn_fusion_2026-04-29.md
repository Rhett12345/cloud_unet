## 1. 最近邻匹配（fusion_core.py）

### 改动前

`_collect_1km()` 使用 `tree.query_ball_point()` 找到搜索半径内所有 MODIS 1km 像元，保留最近的 `EXPECTED_1KM_PER_AGRI`（默认12）个候选。汇总阶段 `aggregate_modis_to_agri()` 对多候选做加权多数表决（Phase）、对数域加权中位数（COT）、加权均值（CER/CTH）。

### 改动后

`_collect_1km()` 改为 `tree.query(k=1, distance_upper_bound=chord)`，每个 AGRI 像元只取最近的单个 MODIS 1km 像元，每个文件至多贡献一个候选。

`aggregate_modis_to_agri()` 汇总逻辑简化为：
- 若有多个文件提供候选，选时间差（dt）最小的那个
- 直接取用该最近邻的 CLP/CER/COT/CTH 值，不做任何聚合
- 质量字段简化：OVERLAP_FRACTION → 0/1，PHASE_CONSISTENCY → 1，CLOUD_FRACTION → 0/1（单像元无覆盖率/一致性概念）
- SAMPLE_WEIGHT = time_weight(best_dt) × 1.0

### 影响的函数

| 函数 | 改动 |
|------|------|
| `_collect_1km()` | `query_ball_point` → `query(k=1)` |
| `aggregate_modis_to_agri()` 汇总循环 | 多候选加权聚合 → 单候选直接取值 |

## 2. MODIS 完整落入 AGRI 圆盘检查

### 新增函数 `check_modis_in_agri_disk()`（fusion_core.py:60-109）

思路：AGRI 圆盘近似为凸形。若 MODIS 条带（矩形）四条边全在圆盘内，则整个条带一定在内部。

实现：
1. 取 AGRI 所有有效（非 NaN）像元坐标 → 构建 KD-tree
2. 采样 MODIS 四条边上的点（每 ~50 个像元取一个）
3. 对每个采样点，在 AGRI KD-tree 中找最近有效像元，计算距离
4. 若任一点距离 > 10km → MODIS 未完整落入 → 返回 False

### 调用位置

`data_fusion.py:_fuse_one_scene()` 中，每读取一个 MODIS 文件后立即检查（第139-145行），未通过则 `continue` 跳过该文件。

优先使用 MYD03 1km 坐标；若不存在则退回到 MYD06 5km 坐标。

## 3. Stats 通道数不匹配修复（dataset.py）

### 问题

```
ValueError: operands could not be broadcast together with shapes (6,) (8,) (6,)
```

`sum_bt` 初始化为 `np.zeros(cfg.AGRI_CHANNELS)`（当前=6），但旧融合文件可能有 8 通道（旧 `AGRI_BT_CHANNEL_INDICES` 含 ch1, ch4, ch8-13）。

### 修复

`_stats_worker()` 两处数据读取路径均添加：

```python
if n_agri != cfg.AGRI_CHANNELS:
    log.warning("Skip %s: BT channels=%d, expected=%d", ...)
    return None
```

当前数据（train 133 + val 35 + test 30 = 198 个文件）全部为 6 通道，校验通过不跳过。若将来混入旧格式文件，会打印 warning 而非崩溃。

## 4. 融合几何可视化脚本

### 功能

- 自动识别 `full_disk`（旧格式 AGRI/BT + Labels/*）和 `samples_v2`（新格式 Samples/agri + geo + labels）
