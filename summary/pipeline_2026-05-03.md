```
main.py ──┬── fuse  ──► data_fusion.py ──► fusion_core.py + fusion_io.py
           │                                     │
           ├── stats ──► dataset.py               ▼
           │                          MYD06 1km ──KD-tree k=1──► AGRI 4km

           │
```

```
python main.py --stages fuse stats train test
```

## 3. Stage 1: fuse — 数据融合

### 3.1 入口与调度

`main.py:stage_fuse()` → `data_fusion.py:fuse_day()`

- 每天作为一个调度单元
- 每个 AGRI 场景文件 = 一个子进程任务 (ProcessPoolExecutor, 默认 16 workers)
- 每天约 40 个 AGRI 场景, 150 个 MYD06, 150 个 MYD03
- 实际产出: 每天 4-8 个 H5 (因 MODIS 完整落入 AGRI 圆盘的时刻有限)

### 3.2 单场景融合流程

`data_fusion.py:_fuse_one_scene()`:

1. **读 AGRI**: `fusion_io.read_agri_scene()` → `dict(lat, lon, VZA, SZA, BT)`
   - BT 通道按 `AGRI_BT_CHANNEL_INDICES` 选择 (8ch)
   - 定标通过 LUT + Slope/Intercept

2. **读 MYD06**: `fusion_io.read_myd06()` → `dict(CLP_1km, CER_1km, COT_1km, CTH_1km, lat_1km, lon_1km, scan_time_1km, ...)`
   - MYD03 提供 1km 经纬度 + 扫描时间
   - **前期 QA 过滤** (`_apply_qa_filter`):
     - Cloud_Mask: CLP 保留全部 (0,1,2,3), CER/COT/CTH 仅保留 (0,3)
     - CER/COT 不确定度 ≤ 100%
     - Optical Phase 要求 (1,2,3,4)
     - CTH 要求有效 CTP/CTT + 允许的 cloud top method
   - 要求 MYD03 1km geo + scan_time (可关闭: `FUSION_REQUIRE_MYD03_1KM=0`)

3. **空间筛选**: `fusion_core.check_modis_in_agri_disk()`
   - 采样 MODIS 条带四边边缘像元, 全部必须在 AGRI 有效像元 10km 内

4. **匹配**: `fusion_core.aggregate_modis_to_agri()`
   - 对每个 MYD06 条带, 构建 MODIS 1km 像元的 KD-tree
   - 每个 AGRI 4km 像元查询 2.5km 半径内最近 1 个 MODIS 像元
   - 多文件候选时选时间差最小的
   - 直接取用该候选的 CLP/CER/COT/CTH 值 (不聚合)

5. **后处理 QC**: `fusion_io.apply_quality_filter()`
   - CLP: `dt ≤ 7.5min` + VZA/SZA ≤ 65°
   - 回归 (CER/COT/CTH): CLP>0 + 值域检查 + `dt ≤ 3.0min` + VZA/SZA ≤ 65°

6. **写出**: `fusion_io.write_fused_samples()`
   - 在 AGRI 全圆盘上滑动 32×32 窗口 (train 50% overlap, val/test non-overlap)
   - 过滤: train ≥ 256 有效 CLP 像元, val/test ≥ 128
   - 格式: `samples_v2` HDF5
     ```
     Samples/agri            (N, 8, 32, 32)   BT通道
     Samples/geo             (N, 4, 32, 32)   lat,lon,VZA,SZA
     Samples/labels          (N, 4, 32, 32)   CLP,CER,COT,CTH
     Samples/max_time_diff_min, mean_sample_weight, ...
     ```

### 3.3 融合配置

`fusion_config.py` — 环境变量可覆盖:

| 参数 | 默认值 | 含义 |
|------|--------|------|
| `TIME_LOW_Q_MIN` | 7.5 min | CLP 最大时间差 |
| `REG_TIME_MAX_MIN` | 3.0 min | 回归最大时间差 |
| `AGRI_SEARCH_RADIUS_KM` | 2.5 km | KD-tree 搜索半径 |
| `REQUIRE_MYD03_1KM` | True | 要求 1km 地理定位 |
| `REQUIRE_SCAN_TIME` | True | 要求像元级扫描时间 |
| `N_FUSION_WORKERS` | 16 | 并行进程数 |

## 4. Stage 2: stats — 归一化统计

`dataset.py:compute_and_save_stats()`

- 并行读取 130 个 train H5 文件 (8 workers)
- 收集 BT 和 label 的 sum/sumsq, 主进程归约
- 计算 mean/std (z-score)
- 回归变量取 5th/95th 百分位 (512K 像元水库)
- 通道数不匹配的 H5 被跳过
- 输出: `norm_stats.npz`

## 5. Stage 3: train — 训练

### 5.1 模型

`model.py:SimpleUNet` — 标准 U-Net with skip connections:

```
输入: agri (8,32,32) + geo (4,32,32) → concat → (12,32,32)

enc1: 12→16      dec0: 32+16→16
enc2: 16→32      dec1: 64+32→32
enc3: 32→64      dec2: 128+64→64
enc4: 64→128     dec3: 256+128→128
bottleneck: 128→256

head: Conv2d(16, 6) → CLP(3) + CER/COT/CTH(3)
```

- 参数量: ~1.97M
- DoubleConv: Conv→BN→ReLU→Conv→BN→ReLU
- 下采样: MaxPool2d(2), 上采样: bilinear Upsample
- geo 通道归一化: lat/90, lon/180, VZA/90, SZA/90

### 5.2 数据集

`dataset.py:AGRIMyd06Dataset` — 懒加载 HDF5:

- `__init__`: 扫描 H5 文件, 构建 patch index (file, i, j)
- `__getitem__`: HDF5 hyperslab 读取单个 patch
  - BT z-score 归一化 + nan→0
  - CLP 标签重映射 (兼容旧 5 类 H5)
  - Label QC: CLP [0,3), CER [0,100], COT [0,200], CTH [0,18000]
  - 回归标签 z-score 归一化, NaN 保留
  - **Train 增强**:
    - 高斯噪声 (σ=0.02)
    - 随机水平/垂直翻转 (p=0.5)
    - 随机 90° 旋转
    - 随机缩放 (0.9-1.1×), reflect pad
    - BT 通道亮度抖动 (p=0.3, σ=0.05)

### 5.3 训练循环

`train.py:train()`:

- **Loss**: CLP(加权CE) + CER(SmoothL1) + COT(SmoothL1) + CTH(SmoothL1), 等权 1.0
- **CLP 类别权重**: sqrt(inverse frequency), 平衡 Clear/Water/Ice
- **优化器**: AdamW (lr=1e-4, weight_decay=1e-3)
- **调度器**: ReduceLROnPlateau (patience=6, factor=0.5, min_lr=1e-6)
- **Early stopping**: patience=10 (按 val_macro_acc)
- **AMP**: torch.cuda.amp GradScaler + autocast
- **Gradient clipping**: max_norm=1.0
- **Batch size**: 32, epochs: 30

## 6. Stage 4: test — 评估

`test.py:evaluate()`:

- 加载 best checkpoint (`val_macro_acc` 最优)
- 全测试集推理, 收集 CLP/CER/COT/CTH 预测
- **CLP 指标**: OA, per-class accuracy, macro accuracy, confusion matrix
- **回归指标**: RMSE, MAE, Bias, R (仅 cloudy + valid 像元)
- 输出: confusion_matrix.png, scatter_{CER,COT,CTH}.png, metrics_summary.csv

## 8. 当前已知问题

1. **CER/COT 标签覆盖率仅 8.1%** — 54.9% 的训练 patch 完全没有回归监督。回归 loss 在大部分样本上被 mask。
2. **单像元匹配的标签噪声** — 1 个 MODIS 1km 像元代表 4km AGRI 像元, 亚像元云异质性被忽略。
3. **CLP 分类区分度低** — Water/Ice recall 通常在 25-30%, 晴空占 40.5%, 模型容易被 "预测 Clear" 的 baseline 支配。
4. **视差修正缺失** — FY-4 静止轨道与 Aqua 极轨的观测几何差异, 高云视差可达数公里。
